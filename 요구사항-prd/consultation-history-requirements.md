# 요구사항 문서: 상담 이력 조회 (Consultation History)

## 1. 개요

### 1.1 목적
권한별 사용자에게 상담 이력(콜) 데이터를 조회·분석·청취할 수 있는 통합 모니터링 환경을 제공한다.

### 1.2 대상 사용자
- 모니터링 사용자, 코치, 팀장, 상담사

### 1.3 권한 정책
| AICC 역할명 | 기존 역할명 | 조회 권한 |
|---|---|---|
| 운영 관리자 | 모니터링 | 홈/모바일 전체 상담 이력 조회 가능 |
| 현업담당자-코치 | 코치 | 본인 소속 서비스 채널의 상담 이력 조회 가능 |
| 현업담당자-팀장 | 팀장 | 본인 소속 서비스 채널의 상담 이력 조회 가능 |
| 현업담당자-상담사 | 상담사 | 본인 소속 서비스 채널 및 본인 상담 이력 조회 가능 |

## 3. 검색 / 필터 영역

### 3.1 영역 설명
- 상담사 정보 및 상담 정보 2개 정보군 기준으로 검색/필터링
- 검색 실행은 [검색] 버튼 클릭 시에만 수행
- **상담 일자는 필수 검색 조건**, 미선택 시 검색 불가

### 3.2 상담사 정보 필터

#### 3.2.1 채널 (필수)
- Type: Dropdown
- 옵션: 전체 / 홈 / 모바일
- 기본값: 전체
- 선택 시 센터 드롭다운 활성화

#### 3.2.2 센터 (필수)
- Type: 체크박스 Dropdown
- 데이터: IM에 등록된 고객센터 상담채널 조직도의 센터 명
- 의존 조건: 채널 선택 후 조회 가능
- 기본값: 전체

#### 3.2.3 그룹
- Type: Dropdown
- 의존 조건: 채널 + 센터 선택 후 조회 가능
- 기본값: 전체

#### 3.2.4 팀
- Type: Dropdown
- 의존 조건: 채널 > 센터 > 그룹 선택 후 조회 가능
- 기본값: 전체

#### 3.2.5 상담사 ID
- Type: Text Input
- 최대 12자, 영문 + 숫자만 입력 가능
- Placeholder: "최대 12자 이내"

### 3.3 상담 정보 필터

#### 3.3.1 상담 방향
- Type: Dropdown
- 옵션: 전체 / I/B (In-bound) / O/B (Out-bound)
- 기본값: 전체

#### 3.3.2 상담 유형
- Type: Dropdown
- 옵션: 업무처리 / 불만 / 이벤트 / 전용코드 / 기타 / TPS 품질조치코드 / TOSS코드

#### 3.3.3 고객 정보 검색
- (1) 고객 정보 유형: Dropdown (인입번호 Default / 가입번호)
- (2) 입력 필드: Text Input, 숫자 또는 영문+숫자 조합

#### 3.3.4 t-NPS 추천 점수
- 연동불가로 고정 '-' 표기

#### 3.3.5 상담 일자 (필수) ✅
- Type: Calendar-based Date Range Picker
- 최소 1일, 최대 7일
- 기본 상태: 선택되지 않은 상태
- 미선택 시 검색 불가, 검색 버튼 Disabled

#### 3.3.6 상담 시간대
- Type: Combo Box (Multi Select)
- 00시 ~ 23시, 중복 선택 가능

#### 3.3.7 통화 시간
- Type: Text
- "SSS초 - SSS초" 형식, 범위 0~9999

#### 3.3.8 상담 코드
- (1) 카테고리: 대 > 중 > 소 Dropdown (종속)
- (2) 코드/키워드 검색: 소카테고리 기준 부분 일치 검색
- 결과: "상담코드 | 소카테고리명" 구조로 노출

## 6. 상담 이력 조회 결과 리스트

### 6.2 기본 정책
- 페이지당 10건
- 좌/우 스크롤 가능
- 전체 조회 건수: "총 {n}건" (천 단위 콤마)
- 검색 전: "검색해주세요"
- 결과 없음: "조회된 상담 이력이 없습니다."
- 컬럼 Max width: 102px (고객정보 제외), 말줄임 + 툴팁

### 6.3 컬럼 구성 (좌→우)

#### 상담 정보
| 컬럼 | 형식 | 정렬 |
|---|---|---|
| 상담일자 | yyyy.mm.dd | ↕ |
| 상담시간 | hh:mm:ss | ↕ |
| 통화시간 | SSS초 | ↕ |
| 상담방향 | IB / OB | ↕ |
| 상담유형 | 업무처리 등 | ↕ |

#### 상담 코드
| 컬럼 | 형식 | 정렬 |
|---|---|---|
| 상담코드 번호 | 전체 자리수 | ↕ |
| 상담코드 카테고리 | 대>중>소 (최대 7글자) | ↕ |

#### 고객 정보
| 컬럼 | 형식 | 정렬 |
|---|---|---|
| 인입번호 | 010-****-1234 | 불가 |
| 가입번호 | ********1234 | 불가 |

#### 상담사 정보
| 컬럼 | 형식 | 정렬 |
|---|---|---|
| 센터 | 센터명 | ↕ |
| 그룹 | 그룹명 | ↕ |
| 팀 | 팀명 | ↕ |
| 상담사 ID | 마당 ID | ↕ |

#### 품질 지수
| 컬럼 | 형식 | 정렬 |
|---|---|---|
| 묵음 횟수 | {n}회 | ↕ |
| 묵음 시간 | {n}초 | ↕ |
| 추천 점수 | - (연동불가) | ↕ |

## 7~13. 상담 이력 상세

### 화면 구성
- 상단 타이틀 (breadcrumb: 상담 이력 조회 > 상담 상세)
- 콜 청취 버튼 (Pop-up 녹취 플레이어)
- 상담 정보 영역 (조직/상담사, 상담 기본, 상담 코드, 고객 정보, 품질 지표)
- STT 영역 (고객: 좌측 버블, 상담사: 우측 보라색 버블, 시간: HH:MM:SS)
- AI 상담 요약 (고객 요청 요약, 상담사 처리 요약)
